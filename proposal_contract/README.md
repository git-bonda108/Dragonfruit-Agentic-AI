# Dragonfruit — Contract proposal package

This folder contains everything needed to submit a **winning proposal** aligned with the [Dragonfruit AI & Automation Execution Brief](https://www.dragonfruitmedia.co/) (slides 1–8): system architecture, AI toolkit, roadmap, costs with buffers, and evaluation-criteria alignment.

## Contents

| File | Description |
|------|-------------|
| **Dragonfruit_Proposal_EOY2026.html** | Full proposal (Dragonfruit color scheme). Includes **“The solution in plain language”** (what you have, what we add, the four layers) for non-technical readers. Open in a browser → **Print → Save as PDF** for submission. |
| **architecture_diagram.svg** | System architecture diagram (triggers → agents → connectors → external systems, with HITL callouts). Vector; use as-is or export to PNG. |
| **architecture_diagram.png** | PNG for the proposal (generated by `build_diagram_png.py` — no Cairo needed). |

## How to produce the PDF

1. **Open the proposal in a browser**  
   Open `Dragonfruit_Proposal_EOY2026.html` in Chrome, Safari, or Edge (double-click or drag into the browser).

2. **Export to PDF**  
   - **Chrome / Edge:** `Ctrl+P` (Windows) or `Cmd+P` (Mac) → Destination: **Save as PDF** → Save.  
   - **Safari:** File → Export as PDF.

3. **Optional: embed the diagram as PNG**  
   If the SVG does not appear in the PDF, run `python export_diagram_png.py` (after `pip install cairosvg`) to create `architecture_diagram.png`, then in the HTML replace the `img` `src` with `architecture_diagram.png` and print again.

## How to get the architecture diagram as PNG

**Option A — Script (recommended, no Cairo)**  
```bash
cd proposal_contract
python build_diagram_png.py
```
Creates `architecture_diagram.png` using matplotlib (only standard Python + matplotlib required).

**Option B — Manual**  
1. Open `architecture_diagram.svg` in a browser (Chrome/Safari/Firefox).  
2. Right-click → Inspect → optional: increase zoom (e.g. 200%).  
3. Take a screenshot of the diagram area, or use a tool (e.g. [CloudConvert](https://cloudconvert.com/svg-to-png)) to convert SVG → PNG.

## Final proposal — everything from the brief

To make the proposal explicitly cover **every** requested item (quick wins, all 16 wish-list tools, and criteria), use **PROPOSAL_FINAL_ADDITIONS.md** and the script **apply_final_proposal_additions.py**. The guide lists what to add (summary section, Quick wins, Complete wish list table, criteria one-liner) and where to paste it; the script applies the same changes automatically when the proposal file is accessible. After applying, the document includes: objective, quick wins (first 30–35 days), 8 high-priority + 8 medium-priority tools, requirements A–D, and evaluation criteria.

## Proposal alignment to the brief

- **Slide 4 (Materials):** Proposal references your time allocations, pipeline, tool stack (~$12.8K), AI survey (73 respondents), and wish list (16 tools).  
- **Slide 5 (Objective):** 4→7 clients per pod by EOY 2026; workflow automation, agentic infrastructure, AI wish list toolkit.  
- **Slide 6–7 (Requirements):**  
  - **A** — System architecture with diagram; automation opportunities; build vs buy; time-saved mapping.  
  - **B** — AI toolkit per role with justification, cost, adoption; **market context** (who else does this: Lindy, UpSys, Evolve, etc.) and our positioning; **per-tool** approach, timeline, maintenance, cost range as in the wish list brief.  
  - **C** — Phased roadmap with buffers, KPIs, risk mitigation (HITL, rollback), training.  
  - **D** — Labour (95–145 days with buffer), **AI tools cost framing** (current $12.8K unchanged; new = implementation in labour + ongoing only); system costs ($3.5–8K/month), engagement structure, team.  
- **Slide 8 (Evaluation):** Section E maps each criterion (30% Specificity, 25% Architecture, 20% Roadmap, 15% Cost, 10% Engagement) with best-in-class responses. **Cost:** Per-tool ranges in Appendix 1; current vs incremental clearly separated.
